## creative code toolbox

### episode 01: #phases

>   #creative-code toolbox aims to master the basics of #p5js

Each tool will pass thru the phases... Not all phases are needed (sometimes research covers it, sometimes we need to build something).

1. **Scope**: feature boundary
2. **Research**: what's already out there? in which state? what can we reuse? (libraries, best practices, experiments)
3. **Product**: what's missing? what to build? what's the MVP?
4. **Development**: hands-on. Divided in *isolated* (mostly [glitch](https://glitch.com/)) and *integrated* (our [repo](https://github.com/nonlinear/play/))
5. **Packaging**: prepping for external use: versioning, documentation, wish list...
6. **Promotion**: Going on tour, demos
7. **Feedback**:  rebuild wish list, idolizing possible collaborators, etc
